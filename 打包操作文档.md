```cmd

```

**Windows PowerShell：**
```powershell
pyinstaller --onedir --name main_exe main.py
```

**Linux/Mac：**
```bash

```

**参数说明：**
- `--onefile`: 打包成单个 exe 文件
- `--name`: 输出文件名
- `--add-data`: 添加数据文件（格式：源文件;目标路径）
- `--hidden-import`: 显式导入隐式引用的模块
- `--noconsole`: 隐藏控制台窗口（GUI 程序使用）
- `--icon`: 设置程序图标（需要 .ico 文件）

## 八、常见问题

### 8.1 打包失败：ModuleNotFoundError

**原因：** PyInstaller 未检测到隐式导入的模块

**解决：**
```bash
# 添加隐式导入
pyinstaller --onefile --hidden-import=模块名 main.py
```

### 8.2 打包后运行报错：找不到文件

**原因：** 数据文件路径问题

**解决：**
```python
# 在代码中使用相对路径处理
import os
import sys

def get_resource_path(relative_path):
    """获取资源文件绝对路径"""
    if hasattr(sys, '_MEIPASS'):
        # PyInstaller 打包后的路径
        return os.path.join(sys._MEIPASS, relative_path)
    # 开发环境路径
    return os.path.join(os.path.abspath("."), relative_path)

# 使用
excel_path = get_resource_path("用例_升级.xlsx")
```

### 8.3 打包文件过大

**原因：** 包含了不必要的依赖

**解决：**
```bash
# 使用虚拟环境，只安装必要的依赖
# 或在 spec 文件中排除不需要的模块
excludes=['tkinter', 'matplotlib', 'numpy'],
```

### 8.4 Playwright 打包后无法运行

**原因：** Playwright 需要浏览器驱动

**解决：**
```bash
# 方式一：在目标机器上安装浏览器
playwright install chromium
```

**Windows CMD（方式二）：**
```cmd
pyinstaller --onefile --add-data "venv\Lib\site-packages\playwright\driver;playwright\driver" main.py
```

**Windows PowerShell（方式二）：**
```powershell
pyinstaller --onefile `
  --add-data "venv\Lib\site-packages\playwright\driver;playwright\driver" `
  main.py
```

### 8.5 杀毒软件误报

**原因：** PyInstaller 打包的 exe 可能被误报为病毒

**解决：**
- 添加杀毒软件白名单
- 使用代码签名证书签名 exe
- 使用 `--debug` 参数重新打包，检查是否有恶意代码

## 十、自动化打包脚本

### 10.1 Windows CMD 批处理脚本

创建 `build.bat`：
```batch
@echo off
REM DPI 安装升级测试工具 - Windows CMD 打包脚本

echo ================================
echo DPI 安装升级测试工具 - 自动打包
echo ================================

echo.
echo [1/5] 清理旧文件...
if exist build rmdir /s /q build
if exist dist rmdir /s /q dist
if exist main_exe.spec del main_exe.spec

echo.
echo [2/5] 激活虚拟环境...
if not exist venv (
    echo 虚拟环境不存在，正在创建...
    python -m venv venv
)
call venv\Scripts\activate.bat

echo.
echo [3/5] 检查依赖...
python -m pip install --upgrade pip
pip install -r requirements.txt
pip install pyinstaller

echo.
echo [4/5] 开始打包...
pyinstaller --onefile --name main_exe --hidden-import=comm --hidden-import=common --hidden-import=dict_comparer --hidden-import=dpi --hidden-import=dpiinstall --hidden-import=dpistat --hidden-import=excel --hidden-import=extract_release_path --hidden-import=ftp --hidden-import=linux --hidden-import=read_write_excel --hidden-import=socket_linux --hidden-import=ssh --hidden-import=xml_comparer --hidden-import=xlwings --hidden-import=paramiko --hidden-import=sshtunnel --hidden-import=ntplib --hidden-import=scapy --hidden-import=scapy.all --hidden-import=playwright --hidden-import=playwright.sync_api --hidden-import=bs4 main.py

echo.
echo [5/5] 验证打包结果...
if exist dist\main_exe.exe (
    echo ✓ 打包成功！
    echo 输出文件：dist\main_exe.exe
    echo.
    echo 测试运行：
    dist\main_exe.exe --help
) else (
    echo ✗ 打包失败！
    pause
    exit /b 1
)

echo.
echo ================================
echo 打包完成！
echo ================================
pause
```

## 十一、持续集成（CI/CD）

### 11.1 GitHub Actions 自动打包

创建 `.github/workflows/build.yml`：
```yaml
name: Build Executable

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: windows-latest

    steps:
    - uses: actions/checkout@v3

    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.10'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
        pip install pyinstaller

    - name: Build executable
      run: |
        pyinstaller --onefile --name main_exe --add-data "用例_升级.xlsx;." main.py

    - name: Upload artifact
      uses: actions/upload-artifact@v3
      with:
        name: main_exe-windows
        path: dist/main_exe.exe
```

# DPI 安装升级测试工具 - 打包操作文档

## 一、环境准备

### 1.1 安装 Python
从官网下载并安装 Python 3.8+：https://www.python.org/downloads/

安装时勾选：
- ✅ Add Python to PATH
- ✅ Install pip

### 1.2 验证安装
```bash
python --version
pip --version
```

---

## 二、创建虚拟环境

### 2.1 创建虚拟环境
```bash
# 进入项目目录
cd E:\PycharmProjects\main_exe

# 创建虚拟环境
python -m venv venv
```

### 2.2 激活虚拟环境

**Windows (CMD):**
```cmd
venv\Scripts\activate.bat
```

**Windows (PowerShell):**
```powershell
venv\Scripts\Activate.ps1
```

**Linux/Mac:**
```bash
source venv/bin/activate
```

### 2.3 升级 pip
```bash
python -m pip install --upgrade pip
```

---

## 三、安装依赖

### 3.1 安装 Python 依赖库
```bash
pip install -r requirements.txt
```

### 3.2 安装 Playwright 浏览器
```bash
playwright install chromium
```

### 3.3 安装打包工具
```bash

```

## 六、部署说明

### 6.1 文件清单
```
dist/
├── main_exe.exe           # 主程序
├── 用例_升级.xlsx         # 测试用例（外部配置文件）
├── versions.json          # 版本配置（外部配置文件）
└── README.txt             # 使用说明
```

### 6.2 配置文件说明
- **用例_升级.xlsx**：测试用例配置文件，必须与 exe 在同一目录
- **versions.json**：版本配置文件，必须与 exe 在同一目录

### 6.3 注意事项
- 配置文件必须与可执行文件在同一目录
- 用户可以直接修改配置文件，无需重新打包
- Playwright 需要浏览器驱动，首次运行可能需要安装：`playwright install chromium`

## 七、常见问题

### 7.1 PowerShell 执行策略错误
```powershell
# 错误：无法加载文件，因为在此系统上禁止运行脚本
# 解决：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 7.2 打包后运行报错：找不到文件
**原因：** 配置文件路径问题

**解决：** 确保配置文件与 exe 在同一目录

### 7.3 打包文件过大
**解决：** 使用虚拟环境，只安装必要的依赖

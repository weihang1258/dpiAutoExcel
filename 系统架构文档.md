# DPI 自动化测试系统架构文档

## 一、系统架构

DPI 自动化测试系统是一个基于 Excel 用例驱动的自动化测试框架，支持 DPI（深度包检测）设备的安装、升级、模式切换等测试场景。

```mermaid
graph TD
    subgraph 用户交互
        USER[👤 测试人员]
    end

    subgraph 输入输出
        EXCEL[📋 Excel 用例]
        REPORT[📊 测试报告]
        LOG[📝 日志文件]
    end

    subgraph 主控程序
        MAIN[main.py<br/>入口 & 调度]
        RW[read_write_excel.py<br/>用例解析]
        EX[excel.py<br/>报告生成]
    end

    subgraph 业务逻辑
        INSTALL[dpiinstall.py<br/>安装/升级/切换]
        FTP[ftp.py<br/>FTP 下载]
        ERP[extract_release_path.py<br/>版本提取]
        LOG[log_handler.py<br/>日志处理器]
    end

    subgraph 设备操作
        DPI[dpi.py<br/>DPI 操作封装]
    end

    subgraph 数据存储
        VJSON[versions.json<br/>版本信息]
    end

    USER -->|编写| EXCEL
    USER -->|执行| MAIN
    MAIN --> RW
    MAIN --> EX
    MAIN --> INSTALL
    INSTALL --> FTP
    INSTALL --> ERP
    INSTALL --> DPI
    INSTALL --> LOG
    ERP --> VJSON
    EX --> REPORT
    MAIN --> LOG
```

## 二、核心业务流程

### 2.1 测试执行总流程

```mermaid
graph TD
    Start([用户执行脚本]) --> A[解析 Excel 用例]
    A --> B[读取测试配置<br/>设备信息/FTP地址]
    B --> C{遍历每个测试用例}
    C --> D{判断安装类型}
    D -->|全新安装| E[执行安装流程]
    D -->|模式切换| F[执行切换流程]
    D -->|版本升级| G[执行升级流程]
    E --> H[记录测试结果]
    F --> H
    G --> H
    H --> I{还有用例?}
    I -->|是| C
    I -->|否| J[生成测试报告]
    J --> End([结束])
```

### 2.2 三种测试场景

#### 场景1：全新安装

```mermaid
graph TD
    Start([开始全新安装]) --> A[获取 FTP 安装包路径]
    A --> B[下载安装包到设备]
    B --> C[解压安装包]
    C --> D[配置 DPI 参数]
    D --> E[启动 DPI]
    E --> F{DPI 是否正常运行?}
    F -->|是| G[✓ 安装成功]
    F -->|否| H[✗ 安装失败]
    G --> End([结束])
    H --> End
```

**用例配置示例：**
- 安装类型：全新安装
- dpiversion_d：目标版本号（如 1.0.6.0-1）
- dpimode_d：目标模式（如 idc31）

#### 场景2：模式切换

```mermaid
graph TD
    Start([开始模式切换]) --> A{当前版本是否<br/>匹配源版本?}
    A -->|匹配| B[直接执行模式切换]
    A -->|不匹配| C[扫描备份目录]
    C --> D{找到版本+模式<br/>匹配的备份?}
    D -->|是| E[恢复备份]
    D -->|否| F[全新安装源版本]
    E --> G[执行模式切换]
    F --> G
    G --> H{模式切换成功?}
    H -->|是| I[✓ 切换成功]
    H -->|否| J[✗ 切换失败]
    I --> End([结束])
    J --> End
```

**用例配置示例：**
- 安装类型：模式切换
- dpiversion_s：源版本号（如 1.0.6.0-1）
- dpimode_s：源模式（如 idc31）
- dpimode_d：目标模式（如 idc32）

#### 场景3：版本升级

```mermaid
graph TD
    Start([开始版本升级]) --> A[安装源版本<br/>同模式切换流程]
    A --> B[备份当前配置]
    B --> C[下载目标版本安装包]
    C --> D{使用升级系统?}
    D -->|是| E[升级系统接管]
    D -->|否| F[本地升级流程]
    E --> G{升级是否成功?}
    F --> G
    G -->|是| H[✓ 升级成功]
    G -->|否| I[✗ 升级失败]
    H --> End([结束])
    I --> End
```

**用例配置示例：**
- 安装类型：升级
- dpiversion_s：源版本号（如 1.0.6.0-1）
- dpiversion_d：目标版本号（如 1.0.6.0-2）
- dpimode_s：源模式（升级不改变模式）

## 四、版本管理机制

### 4.1 versions.json 文件结构

```json
{
  "信息安全执行单元": {
    "网络安全执行单元V1.0.6.0": {
      "1.0.6.0-1": [
        "ftp://10.128.5.116/.../ACT-DPI-ISE-1.0.6.0-1.tar.gz",
        "ftp://10.128.5.116/.../dev-dpi-1.0.6.0-1.tar.gz"
      ],
      "1.0.6.0-2": [...]
    }
  }
}
```

### 4.2 版本路径自动更新流程

```mermaid
graph TD
    Start([用例需要版本 X]) --> A{在 versions.json<br/>中查找版本 X}
    A -->|找到| B[返回 FTP 路径]
    A -->|未找到| C[自动触发更新]
    C --> D[从 RDM 平台<br/>提取版本信息]
    D --> E[更新 versions.json]
    E --> F{再次查找版本 X}
    F -->|找到| G[返回 FTP 路径]
    F -->|未找到| H[报错退出]
    B --> End([结束])
    G --> End
    H --> End
```

### 4.3 模式到分类的映射

```mermaid
graph LR
    A[DPI 模式后缀] --> B[分类名称]
    B --> C[is → 信息安全执行单元]
    B --> D[isbns → 信息安全执行单元]
    B --> E[bns → 网络安全执行单元]
    B --> F[ns → 网络安全执行单元]
    B --> G[bnsns → 网络安全执行单元]
    B --> H[ds → 数据安全执行单元]
```

**示例：**
- `com_cmcc_is` → 信息安全执行单元
- `com_cucc_isbns` → 信息安全执行单元
- `com_ctcc_bns` → 网络安全执行单元

## 五、关键功能模块

### 5.1 DPI 操作类 (dpi.py)

```mermaid
classDiagram
    class Dpi {
        +install() 安装 DPI
        +uninstall() 卸载 DPI
        +upgrade() 升级 DPI
        +start() 启动 DPI
        +stop() 停止 DPI
        +restart() 重启 DPI
        +mod_switch() 模式切换
        +get_dpimode() 获取当前模式
        +get_dpiversion() 获取版本号
        +modify_xsajson() 修改 xsa.json
        +modify_modcfg() 修改模块配置
        +config_pcicfg() 配置 PCI
        +wait_alive() 等待 DPI 启动
        +check_status() 检查运行状态
    }
```

### 5.2 模式切换智能判断 (mod_switch)

```mermaid
graph TD
    Start([调用 mod_switch<br/>mode='idc31']) --> A[获取当前模式]
    A --> B{当前模式 == 目标模式?}
    B -->|是| C{检查 pcicfg<br/>是否变化?}
    B -->|否| D[执行模式切换]
    C -->|有变化| D
    C -->|无变化| E[跳过切换<br/>输出: 无需切换dpi模式]
    D --> F{切换成功?}
    F -->|是| G[✓ 切换成功]
    F -->|否| H[✗ 切换失败]
    G --> End([结束])
    H --> End
    E --> End
```

**关键特性：**
- ✅ 自动检测当前模式
- ✅ 模式匹配时智能跳过
- ✅ 支持强制切换（force=True）
- ✅ 支持配置参数变更触发切换

## 六、执行方式

### 6.1 命令行执行

```bash
# 执行所有 Sheet
python main.py -f 用例_升级.xlsx

# 执行指定 Sheet
python main.py -f 用例_升级.xlsx -s install

# 生成执行脚本
python main.py -bat  # 生成 .bat 脚本
python main.py -ps1  # 生成 .ps1 脚本
```

### 6.2 脚本执行

## 十、注意事项

### 10.1 版本管理

- ✅ `versions.json` 会自动更新，无需手动维护
- ✅ 版本提取使用系统浏览器（Chrome/Edge），无需额外安装 Chromium
- ✅ 支持多项目版本管理

### 10.2 备份恢复

- ✅ 优先使用备份可加快测试速度
- ✅ 备份目录需包含 `ver.txt` 文件（版本信息）
- ✅ 模式匹配时才会使用备份

### 10.3 模式切换

- ✅ 自动检测当前模式，避免重复切换
- ✅ 支持配置参数变更触发切换
- ✅ 切换失败会自动回滚

## 十一、日志系统架构

### 11.1 日志系统设计

```mermaid
graph TD
    subgraph 日志输出
        A[各模块 logger] --> B[ConsoleHandler<br/>控制台输出]
        A --> C[FileHandler<br/>模块日志文件]
        A --> D[DynamicFileHandler<br/>用例日志文件]
    end
    
    subgraph 日志文件
        C --> E[log/common.log<br/>通用工具日志]
        C --> F[log/dpi.log<br/>DPI 操作日志]
        C --> G[log/ftp.log<br/>FTP 传输日志]
        
        D --> H[log/会话ID_sheet_用例.log<br/>用例独立日志]
    end
    
    subgraph 日志格式
        I[统一格式] --> J['[时间] [模块名] [级别] 消息']
    end
```

### 11.2 DynamicFileHandler 工作机制

```mermaid
graph LR
    Start([Sheet 开始]) --> A[创建 DynamicFileHandler]
    A --> B[添加到所有模块 logger]
    B --> C{遍历用例}
    
    C --> D{按用例拆分?}
    D -->|是| E[切换到用例日志文件]
    D -->|否| F[使用 Sheet 日志文件]
    
    E --> G[执行用例]
    F --> G
    
    G --> H[日志输出到当前文件]
    H --> I{还有用例?}
    I -->|是| C
    I -->|否| J[关闭 DynamicFileHandler]
```
